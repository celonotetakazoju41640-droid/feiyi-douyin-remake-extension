# 飞蚁抖音复刻

这是一个 Chrome 扩展版的短视频复刻工作台，目标不是一键直出成片，而是先把参考短视频拆成可执行的复刻结构，再整理成提示词、镜头表和批量任务。

当前仓库重点解决 4 件事：

1. 沉淀多个对标账户模板，而不是每次从零写提示词。
2. 支持抓取 TikTok 公开主页样本，做账号蒸馏和样本筛片。
3. 把复刻结果整理成复刻包、Clipcat 指令、批量视频任务和导出文件。
4. 给 TikTok / 抖音双平台打通统一入口，并支持公开主页样本蒸馏。
5. 在蒸馏管理里补一层“本地文件夹视频深蒸馏”，单独沉淀 0 帧起手、情绪曲线、镜头节奏和商品露出结构。

如果你是刚接手这个仓库，建议先看这份说明，再看路线文档：

- [ROADMAP.md](./ROADMAP.md)

## 平台边界

### TikTok

- 支持公开主页样本自动蒸馏。
- 支持基于主页样本做筛片、重蒸馏和摘要回填。
- 需要扩展拥有 `https://www.tiktok.com/*` 页面权限。

### 抖音

- 已支持公开主页样本级别的自动蒸馏。
- 当前已从最小可用版推进到第二层增强：抓公开主页样本、回填模板、优先补标题 / 封面 / 时长 / 互动数据，并复用现有筛片台。
- 在真实可见卡片存在时，当前还能直接补到：
  - 卡片封面图
  - 点赞数
  - 去重后的标题文案
- 还不支持登录态抓取、私有内容和深翻账号详情。

## 使用流程

### 1. 打开工作台

从扩展弹窗点击“打开复刻工作台”。

### 2. 左侧栏目

- `生成`：选模板、传图、生成项目
- `历史记录`：先选项目，再看详情和导出
- `蒸馏管理`：维护模型、提炼主页、筛样本
  - 同时承接“本地文件夹视频深蒸馏”

### 3. 生成

- 首屏现在只保留一个主任务卡，按最短路径完成：
  - 选择蒸馏模型
  - 上传产品图
  - 等系统自动识别卖点、场景和提示词草稿
  - 点击生成项目
- 需要时再展开：
  - 更多设置
  - 故事版与角色
  - 主页参考
- 如果当前选到的是很空的新模板，页面会先提示你补模板；不想现在补，也可以先继续填。
- 输出设定会继续保留这些字段：
  - 蒸馏模型
  - 发布平台
  - 视频比例
  - 生成时长
  - 口播语言
  - 生成数量
- 主页参考现在收成一条辅助输入，不再单独占很高的首屏区域；不填也能继续。
- 平台语言继续自动判断：TikTok 默认英文，抖音默认中文。
- 首次打开时建议先看一遍 `开始前先看`：
  - 产品图要单独传
  - 有现成脚本时要写明 `不要改动脚本`
  - 历史记录页给的是提示词任务和提交入口，不是直接成片

### 4. 蒸馏管理

- `主页快蒸馏`：
  - 贴 TikTok / 抖音公开主页链接
  - 点击提炼主页模型
  - 前台会显示进度、阶段和失败原因
- `视频深蒸馏`：
  - 直接选择本地视频文件夹
  - 当前先读取文件名、相对路径、时长、大小和修改时间
  - 支持点击 `自动分析当前视频`
  - 会先在前台抽多帧，再交给本地服务走谷歌视觉模型分析
  - 分析结果会自动进入生成链路，不需要用户再手动搬运
  - 然后手动补每条视频的：
    - 0 帧起手
    - 商品强露出秒点
    - 钩子类型
    - 情绪曲线
    - 镜头节奏
    - 卖点证明
    - 收口方式
    - 场景推进
    - 视觉 DNA
    - 一句话总结
- 当前不会上传本地视频，也不会在这一版直接调用真实外部视觉模型。

### 4. 上传图片

- 产品图必填，上传后会自动拆一版商品名、卖点和提示词草稿。

### 5. 生成项目

- 只需要补：
  - 商品名
  - 生成条数
  - 创作要求
- 自动拆出的卖点默认折叠，需要时再展开调整。
- 生成页现在会直接显示 `主流程状态`：
  - 会告诉你蒸馏模型、商品图、生成项目、故事版、带走结果这 5 步现在各自到哪了
  - 按钮不能点时，提示语会和按钮禁用原因保持一致，不再出现“看起来能做、实际还差一步”的情况
  - 默认模板已经存在时，首页不会再误提示“先上传商品图”，而是直接告诉你“再上传商品图就能生成”
  - 如果项目已经生成完成，顶部会优先明确告诉你“项目已生成”，不会再被上一轮商品图识别提示盖过去
- 点击 `生成项目` 后，会自动切到 `历史记录` 查看结果。

### 6. 历史记录

- 首屏先展示项目卡片库。
- 每张卡现在只保留短摘要、少量标签和查看详情按钮，不再在首屏摊长文。
- 历史卡片会直接显示商品图缩略图和生成时间，方便快速判断。
- 选中项目后，再展开详情工作台。
- 如果项目勾选了故事版，详情页里的 `故事版图` 标签现在是一次点击自动提交并等待结果，不需要再手动刷新状态。
- 故事版处理中不再只靠一闪而过的提示：
  - 详情区会常驻显示当前故事版进度
  - 左侧当前项目摘要里也会保留最近一次故事版状态
- 故事版图生成完成后，当前可以直接下载单张图片或一键下载全部故事版图片，不再只给图片链接。
- 结果区现在也会直接给故事版主按钮，不需要用户自己再去找 `故事版图` 标签：
  - 未生成时直接显示 `生成故事版图`
  - 处理中时显示 `继续等待故事版`
  - 已完成时直接显示 `下载故事版图片`
- 故事版任务开关现在默认就是开启状态，普通用户按主流程生成后，后面可以直接接故事版，不需要先自己想到去勾这个选项。
- 商品图上传后，当前会优先走本地视觉分析服务，尽量按图片真实内容提炼商品名、卖点、场景和角色草稿；只有服务不可用时，才退回文件名和模板规则兜底。
- 商品图这轮自动识别一完成，页面会先保留“已自动提炼商品名、卖点、场景和提示词草稿”的反馈，不会在同一次上传里立刻又跳回“可以直接生成”，避免你看不出 AI 到底有没有先识别完。
- 主流程状态第 2 步现在也会常驻保留这次商品图识别结果：会明确写出是 `已识别完成`，还是 `已按文件名兜底提炼`，不再只靠一闪而过的提示。
- 生成前的上传卡现在也会直接给一块 `商品图识别摘要`，把 `识别状态 / 商品名 / 主卖点 / 主场景 / 角色结构` 前置展示出来，不用再自己展开字段核对。
- 这个识别结果现在也会跟项目一起进历史记录和导出的 JSON；你后面切回历史项目时，仍然能看出当时是图片识别成功，还是兜底提炼。
- 如果你先切了平台模式，再马上上传商品图，系统现在会先同步到当前平台优先模板，再去做自动识别，避免抖音模式还沿用 TikTok 模板上下文，或反过来错位。
- 现在如果一次上传多张商品图，系统会把多张图一起送进本地视觉分析服务，不再只看第一张；补充视角图、使用图、细节图会一起参与卖点和场景判断。
- 如果你换了一批商品图重新上传，系统现在会刷新上一轮“自动填出来”的商品名、卖点、场景、提示词草稿和角色草稿；但如果这些字段是你手动改过的，会尽量保留，不会粗暴覆盖。
- 项目生成后刷新页面，或从历史记录里切回项目时，状态区现在也会继续识别这个项目曾上传过几张商品图，不会因为浏览器文件框清空了，就误显示成“商品图未上传”。
- 切换历史项目时，商品图状态条也会立刻跟着当前项目刷新，不会再短暂残留上一个项目的上传状态。
- 切换历史项目时，本地商品图文件输入和预览现在也会一起清空，避免项目 B 误继续沿用项目 A 残留的图片文件。
- 向导区底部的“生成并带走”按钮现在也和主生成按钮共用同一套门槛：没有当前会话新上传的商品图，就不会从这里绕过限制直接重新生成。
- 对于已经恢复出来的历史项目，向导里切到第 3 步、第 4 步只是查看结构时，也不会再误提示“还没上传商品图”；只有真正开始新一轮生成时，才继续要求重新上传当前项目要用的图片。
- 批量导出项目现在不只导出文本和链接，也会顺带下载故事版图片文件本体。
- 结果区现在有真正的主动作：
  - 故事版没齐时，直接提示 `先生成故事版再带走`
  - 结果齐全时，直接用 `一键带走全部结果` 完成复制结果包、导出文件和下载图片
- 如果故事版能在当前轮询里完成，`一键带走全部结果` 现在会自动继续后面的复制和导出，不需要用户再点第二次。
- 如果故事版这轮还没跑完，`一键带走全部结果` 现在也不会静默停住：会明确提示“自动带走先暂停”，告诉你稍后继续等待，或再点一次同一个按钮继续后面的复制和导出。
- `一键带走全部结果` 跑完后，左侧摘要会保留完成态：
  - 会明确告诉你结果包是否已复制
  - 文本文件是否已导出
  - 故事版图片成功下载了多少张
- 如果一键带走中途失败，前台现在也会明确报出 `结果整理失败：...`，不会只把按钮恢复却不告诉你失败在哪一步。
- 如果只是剪贴板复制失败，一键带走现在也不会直接中断；文本导出和故事版图片下载还会继续跑，再把“复制失败但导出继续完成”的结果明确告诉你。
- 生成页现在也给了默认主动作 `生成并一键带走结果`；如果只想先生成项目再看结构，旁边还保留 `只生成项目`。
- 生成页在真正点主按钮前，现在也会直接显示 `任务数 / 比例 / 故事版是否开启 / 主按钮后续动作`，用户能先看清这次会生成几条任务，以及系统会不会继续自动接故事版和一键带走。
- 左侧保留当前项目摘要和高频动作：
  - 提交生成
  - 复制结果包
  - 下载 Markdown（已带故事版结果）
  - 下载 JSON
- `复制结果包` 现在不只带任务和图片链接，也会把视频提示词、关键帧提示词、候选版本摘要一起打进去，更接近真正可直接带走的交付包。
- 右侧 `项目详情 -> 提示词` 现在也会把视频提示词和关键帧提示词一起展示出来，不用非得先导出或复制结果包，前台就能直接检查两层提示词。
- `复制结果包` 里的视频任务段现在还会带上 `任务 ID / 批次号 / 参考链接 / 附加要求`，方便跨电脑继续追任务，或直接交给别的同事接力。
- `下载 Markdown` 和导出包里的 `06-批量视频任务.md` 现在也同步带上 `批次号 / 参考链接 / 附加要求`，三个结果出口的任务追踪信息已经统一。
- `提交生成` 成功后，历史卡片和当前项目状态现在会真正切到 `已提交`，不再只是弹一下批次号提示。
- `提交生成` 按钮本身也会进入提交后状态，显示 `已提交到本地服务` 并禁用，避免用户连续误点重复提交。
- 当前项目摘要区现在也会常驻显示 `当前批次号`，刷新后只要项目里已回写批次号，仍能继续追踪。
- 提交成功后，摘要和下一步建议现在也会把两条去向拆开说清楚：视频任务已经进入本地服务阶段继续跑，文案和图片仍然可以继续一键带走。
- 提交成功后，前台现在也会明确告诉你：可以按 `当前批次号` 去本地服务或外部视频工具继续追踪视频任务，不再只给一串批次号让你自己猜怎么用。
- 历史状态判断现在不只认 `queued`，只要项目里已有 `batchId`，或任务状态是 `queued / submitted / running / processing`，前台都会继续识别成 `已提交`。
- 历史卡片顶部状态现在也不再只剩 `可提交 / 已提交`：如果故事版还没跑，会直接显示 `待故事版`；故事版处理中会显示 `故事版中`；故事版做完但还没整理结果，会显示 `待带走`；一键带走部分完成会显示 `部分带走`；一键带走完全收齐后才显示 `已带走`。
- 如果一键带走只成功了一部分，当前项目摘要里的 `下一步建议` 现在也会直接说清楚：重新点一次 `一键带走全部结果` 去补复制或补下载；如果视频任务已经提交，还会继续提醒你按批次号追踪视频任务。
- 历史卡片和当前结果摘要现在都会直接显示这次项目绑定的 `模板 / 模型`，不用再点开任务详情猜系统到底按哪个复刻模型在跑。
- `当前故事版进度` 和 `当前带走结果` 这两类常驻状态现在也会跟项目一起保存，刷新或重新打开后不会直接丢失。
- 现在把导出的 JSON 项目再导回工作台时，系统会保留原来的 `batchId`、提交态、故事版任务状态、图片链接和当前流程摘要，不会再被重新生提示词的过程洗回初始状态。
- 批量导出项目现在会额外带上故事版任务说明、任务 JSON 和图片链接清单。
- 右侧是当前项目详情标签页。
- 更细的导入、本地服务、批量导出和镜头微调都收在折叠区。
- 折叠区现在会先露出轻工具和服务状态，再往下才是镜头微调，减少一展开就很重的感觉。

### 7. 蒸馏管理

- 如果你要维护模型、蒸馏主页、筛样本、上传参考视频或复制 Clipcat 指令，再进 `蒸馏管理`。
- 首屏现在收成两个主工具块：
  - 模型与主页提炼
  - 视频深蒸馏
- 主页链接现在可以直接在首屏填写，填完后直接点 `提炼主页模型` 即可。
- 提炼成功后，首屏会先提示本次提炼出了多少样本、多少条带封面和播放。
- 视频深蒸馏也继续保留在首屏，但和主页提炼分成单独一块，不再互相抢层级。
- 深层规则、参考控制和样本筛片继续放在折叠区。
- 普通用户不需要先理解这些能力，也可以直接完成生成。
