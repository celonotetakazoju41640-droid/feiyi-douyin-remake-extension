# 2026-07-15 一条龙结果打包闭环

## 用户原始需求摘要

用户要的不是单点功能，而是完整一条龙：

1. 选择蒸馏模型
2. 一键传图
3. AI 自动识别产品卖点和场景
4. 生成提示词
5. 一键点击生成叙事图
6. 复制提示词和图片，直接带走使用

这轮要求继续朝这个总目标推进，并把设计细节考虑到位。

## AI 对需求的理解

当前主生成链路已经基本可用：

- 选模板存在
- 上传产品图后会自动回填商品名、卖点、提示词草稿、场景和角色默认值
- 可以生成项目
- 可以一键触发故事版图任务并自动等待结果

但“最后一公里”还没闭环：

- 复制动作现在只复制 `batchVideoTasks` JSON
- Markdown 导出没有把故事版图结果带出去
- 批量导出包没有包含故事版图任务说明、图片链接或图片清单

这会导致用户虽然生成完成了，但最后还要自己二次整理，仍然不算真正的一条龙。

## 当前假设

- 本轮不改蒸馏算法本身，也不改前半段生成逻辑。
- 当前最该补的是“结果带走”闭环，而不是再做新的生成能力。
- 故事版图片当前来源仍是远端 URL，本轮先把结果说明和图片链接打包出去，不额外引入图片下载代理。
