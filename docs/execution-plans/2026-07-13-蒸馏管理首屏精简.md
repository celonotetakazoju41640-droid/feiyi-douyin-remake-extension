# 2026-07-13 蒸馏管理首屏精简

## 用户原始需求摘要

用户确认继续，希望蒸馏管理再精简一层，把普通用户不该先看到的复杂字段继续往后藏。

## 我对本次任务的理解

当前蒸馏管理虽然已经分层，但首屏还是信息偏多。  
本轮要把它收成：

1. 首屏只保留常用动作
2. 模型基础字段收进折叠区
3. 当前模型切换入口直接放在蒸馏管理首屏

## 当前假设

- 普通用户最常用的是：选模型、贴主页、提炼、选本地视频、自动分析、保存
- 复杂字段如结构拆解、表达 DNA、样本数量、默认模型，不需要首屏直接展开

## 计划修改范围

- `src/workspace.html`
- `src/workspace.css`
- `src/workspace.js`
- `README.md`
- 本执行计划文档

## 明确不修改的范围

- 不改蒸馏逻辑
- 不改深蒸馏自动分析接口
- 不改生成页和历史记录页

## 执行步骤

1. 给蒸馏管理首屏补当前模型选择入口
2. 首屏保留常用动作
3. 把模型基础字段收进折叠区
4. 补切换同步逻辑
5. 更新 README 和计划文档
6. 跑检查

## 验收标准

- 蒸馏管理首屏能直接切换当前模型
- 首屏不再直接铺开大段模型基础字段
- 普通用户一进来先看到的是主页提炼和视频深蒸馏
- `node --check src/workspace.js` 通过
- `git diff --check` 通过

## 风险和需要用户确认的事项

- 本轮只做展示精简，不改变模板字段本身

## 执行后的实际结果

- 蒸馏管理首屏已补“当前模型”选择入口
- 首屏文案已改成“常用操作”
- 模型基础字段已收进“展开模型字段”
- 首屏默认只保留：
  - 当前模型
  - 主页链接提炼
  - 视频深蒸馏
  - 保存 / 新建 / 删除
- 生成页和蒸馏管理页的模型切换保持同步
- README 已同步说明首屏只保留常用操作

## 验证方式

- 打开蒸馏管理页
- 检查首屏只保留常用动作
- 检查模型切换是否同步
- 运行：
  - `node --check src/workspace.js`
  - `git diff --check`
