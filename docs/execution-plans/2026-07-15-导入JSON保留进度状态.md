# 2026-07-15 导入 JSON 保留进度状态

## 用户原始需求摘要

用户要的是整条链路闭环，不只是当场跑得通，还要能把项目保存、带走、再导回来继续接着做。

## AI 对需求的理解

代码审计发现 `importJsonProject()` 当前会先执行：

- `regeneratePrompts(parsed)`

这会把导入包里原本已经存在的这些状态冲掉：

- 提交状态
- `batchId`
- 故事版任务状态 / 图片链接 / 错误信息
- `workflowStatus`

结果就是：导回一个已经跑过一半的项目，前台会把它看成近似全新项目。

## 当前假设

- 不触发真实外部服务。
