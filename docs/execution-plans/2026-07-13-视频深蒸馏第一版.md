# 2026-07-13 视频深蒸馏第一版

## 用户原始需求摘要

用户确认要开始做“视频深蒸馏”，并明确指出：

- 不能只靠主页快蒸馏
- 很多商品是 0 帧起手
- 关键不只是文案，还要理解画面情绪、镜头推进和商品露出方式
- 不需要上传视频，直接读取本地文件夹里的视频即可

## 我对本次任务的理解

这次要在现有“主页快蒸馏”旁边，补一层“视频深蒸馏”第一版工作台。

第一版目标不是立刻做全自动完整看片，而是先把这条真实链路搭出来：

1. 读取本地文件夹里的视频
2. 在前台列出待深蒸馏的视频样本
3. 定义深蒸馏结构字段
4. 为后续接本地服务 + 视觉模型预留稳定入口

## 当前假设

- 机器已安装 `ffmpeg`
- 现有本地服务脚本可读到 `GEMINI_API_KEY`
- 第一版先不强行接完整的自动视频理解闭环，而是优先把前台入口、数据结构和任务承接方式做稳
- 深蒸馏应与主页快蒸馏分开命名、分开入口、分开结果区

## 计划修改范围

- `src/workspace.html`

- `src/workspace.js`
- `src/remake-core.js`

## 明确不修改的范围

- 主页快蒸馏抓取逻辑

- 生成页主流程
- 真实付费视频生成

## 执行步骤

1. 给模板数据结构补“视频深蒸馏”字段
2. 在蒸馏管理页新增“视频深蒸馏”入口区
3. 支持选择本地文件夹里的视频并列出样本
4. 先沉淀深蒸馏字段框架：
   - 是否 0 帧商品起手
   - 商品首次强露出时点
   - 前 3 秒钩子类型
   - 情绪曲线
   - 镜头节奏
   - 卖点证明方式
   - 收口方式
5. 补说明文档，明确区分“主页快蒸馏 / 视频深蒸馏”
6. 跑检查并做页面核对

## 风险和需要用户确认的事项

- 如果后续接 Gemini 或其他视觉模型，需要继续补本地服务接口
- 本轮不触发真实外部视频生成
