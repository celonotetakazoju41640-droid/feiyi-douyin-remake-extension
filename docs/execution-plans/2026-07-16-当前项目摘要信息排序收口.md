# 2026-07-16 当前项目摘要信息排序收口

## 用户原始需求摘要

用户要把“选择蒸馏模型 -> 一键传图 -> AI 自动识别产品卖点场景 -> 生成提示词 -> 一键点击生成叙事图 -> 复制提示词图片一条龙”这条主链路做成真正傻瓜式，不只是功能能跑，还要让完成页一眼看懂现在最该做什么。

## AI 对需求的理解

当前历史区的“当前项目摘要”虽然信息已经很全，但仍然存在一个完成页体验缺口：

- `主卖点`
- `商品图识别`
- `当前批次号`
- `本地服务阶段`
- `故事版`
- `当前故事版进度`
- `当前带走结果`
- `下一步建议`
- `结果总览`

这些信息现在基本按并列行往下堆。
结果是最关键的两件事：

- 现在处于什么阶段
- 下一步最该点什么

没有被抬到最前。

## 当前假设

- 本轮不改功能逻辑，只收摘要排序。
- `下一步建议` 和当前最关键的阶段状态，应该优先出现在摘要最前面。
- 其他解释性信息继续保留，但后置到第二层。

4. 跑相关检查命令。
