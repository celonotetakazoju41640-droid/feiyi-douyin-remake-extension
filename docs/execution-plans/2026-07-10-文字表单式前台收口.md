# 2026-07-10 文字表单式前台收口

## 用户原始需求摘要

用户明确要求把当前前台从“大块工作台”改成更短、更轻的入口页：

- 不要卡片式工作台
- 不要长页面
- 不要把蒸馏过程摊给普通用户看
- 主界面只保留：
  - 选择蒸馏模型
  - 上传图片
  - 生成提示词

用户还明确要求：

- 蒸馏模型管理单独放一起
- 主界面尽量用文字表单，不要复杂视觉块

## 我对本次任务的理解

这次不是继续优化现有 4 步向导，而是把主界面彻底收成短入口：

- 顶部保留品牌和少量状态
- 中间主区域只做文字表单
- 把普通用户流程压成一屏内可理解的最短链路
- 蒸馏模型管理、主页蒸馏、样本筛片等能力退到独立区域，不进入主流程

## 当前假设

- 继续复用现有 `accountTemplates` 作为“蒸馏模型”数据来源
- 继续复用现有提示词生成逻辑，不改蒸馏算法、不改本地服务协议
- 这轮只改 Chrome 扩展前台呈现和交互组织

## 计划修改范围

- `src/workspace.html`
- `src/workspace.css`
- `src/workspace.js`
- `tests/extension-shell.test.mjs`
- `README.md`

## 明确不修改的范围

- `src/remake-core.js`
- 主页蒸馏算法
- 真实 API 提交
- 权限、计费、额度、后台服务

## 执行步骤

1. 拆掉当前左侧步骤栏和右侧操作坞的主流程结构
2. 把主界面改成短文字表单：模型、平台、图片、要求、生成
3. 把结果区收成更轻的同页文字结果
4. 把蒸馏模型管理和高级能力单独放到折叠区
5. 同步更新测试和 README，再跑检查
