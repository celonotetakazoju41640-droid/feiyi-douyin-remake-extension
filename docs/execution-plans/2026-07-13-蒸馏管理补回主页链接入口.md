# 2026-07-13 蒸馏管理补回主页链接入口

## 用户原始需求摘要

用户反馈蒸馏管理里没办法添加主页链接，导致不知道怎么开始蒸馏。

## 我对本次任务的理解

这次除了补回蒸馏管理页的蒸馏入口，还要把“抓取成功后的结果反馈”补完整，避免用户抓完后第一眼看不出页面哪里变了。

## 当前假设

- 蒸馏逻辑本身还在，前一个问题是蒸馏管理页缺少可直接填写主页链接的入口
- 当前主页链接输入框只保留在生成页的“可选主页参考”
- 当前更大的体验问题是：抓取成功后的结果主要落在折叠区和静默字段回填里，首屏没有足够强的成功反馈
- 只要把蒸馏入口补回蒸馏管理，并把结果摘要和样本区前置一层反馈，就能恢复可感知性

## 计划修改范围

- `src/workspace.html`
- `src/workspace.css`
- `src/workspace.js`
- `README.md`
- 本执行计划文档

## 明确不修改的范围

- 蒸馏算法
- 样本筛片逻辑
- 生成页主流程
- 本地服务和提交链路

## 执行步骤

1. 在蒸馏管理页补回主页链接输入区
2. 让蒸馏管理页和生成页的主页链接输入保持同步
3. 补一块蒸馏结果摘要到蒸馏管理首屏
4. 抓取成功后自动展开“主页蒸馏样本和筛片”
5. 把首屏摘要继续升级成“进度条 + 步骤列表”状态栏
6. 保持现有“提炼主页模型”按钮逻辑不变
7. 更新 README 和执行计划文档
8. 跑检查并做页面核对

## 验收标准

- 用户能在蒸馏管理页直接填写主页链接
- 填完后可以直接点击“提炼主页模型”
- 生成页和蒸馏管理页的主页链接内容保持一致
- 抓取成功后，蒸馏管理首屏会出现明确的结果摘要
- 抓取中时，蒸馏管理首屏会出现明确的进度条和步骤状态
- 抓取成功后，主页蒸馏样本区会自动展开
- `node --check src/workspace.js` 通过
- `git diff --check` 通过

## 风险和需要用户确认的事项

- 本轮只补前台入口，不改真实外部抓取能力边界
- 不会触发真实视频生成或额外外部成本
