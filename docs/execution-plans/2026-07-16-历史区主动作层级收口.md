# 2026-07-16 历史区主动作层级收口

## 用户原始需求摘要

用户要把“选择蒸馏模型 -> 一键传图 -> AI 自动识别产品卖点场景 -> 生成提示词 -> 一键点击生成叙事图 -> 复制提示词图片一条龙”这条主链路做到真正傻瓜式，细节也要考虑到位。

## AI 对需求的理解

当前历史区虽然已经有：

- 提交到本地服务
- 一键带走全部结果
- 生成故事版图

但在不少状态下，`提交到本地服务` 和 `一键带走全部结果` 会同时是高亮主按钮。
这会让普通用户还要自己判断：

- 现在先提视频任务
- 还是先一键带走
- 还是先补故事版

这不符合“完成页默认给一个最该点动作”的目标。

## 当前假设

- 本轮不改任何底层功能逻辑，只收按钮层级。
- 历史区同一时刻应该只突出一个主按钮。
- 优先级应按当前真实主链路走：
  1. 如果此刻应该先一键带走，就突出 `一键带走全部结果`
  2. 如果此刻应该先补故事版，就突出 `生成故事版图 / 继续等待故事版`
  3. 只有前两者都不是当前主动作时，才突出 `提交到本地服务`
