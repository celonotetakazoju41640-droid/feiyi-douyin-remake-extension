# 2026-07-09 C 端首页流程简化执行计划

## 用户原始需求摘要

用户已加载扩展，反馈 UI 有点乱，希望把蒸馏结果和无用功能隐藏掉，首页只保留最核心业务流程：上传素材、补一句要求、生成提示词，并把参考主页、蒸馏能力和历史细节都后置，整体更像面向 C 端用户的网站。

## AI 对需求的理解

这轮不是补新蒸馏算法，而是把现有工作台收口成一套更简单的业务流程。核心目标：

- 首页默认只露出主任务输入，不再堆后台细节
- 主页链接改成可选参考，不再占首页主路径
- 提供产品图、模特图、提示词、提示词条数这些 C 端常用输入
- 历史记录保留，但蒸馏明细、筛片和服务配置收进二级区域

## 执行步骤

1. 先核对当前页面结构和核心流程依赖。
2. 写失败测试，锁定简化版首页必须存在的核心入口。
3. 先把首页从后台长页收成面向普通用户的主流程，再继续往向导式结构演进。
4. 补模特图上传、主页扫描自动入模板列表、简化后的文案反馈。
5. 跑语法、测试和浏览器页面核查。

## 验收标准

- 首页默认显示面向普通用户的简化主流程，后续继续收口为 4 步向导：
  - 基础信息
  - 上传素材
  - 生成要求
  - 结果与提交
- 蒸馏结果、筛片、批量任务、本地服务这些功能不再常驻首页主视觉。
- 页面存在模特图上传入口。
- 主页扫描完成后，模板会自动加入模板下拉列表。
- `node --check src/workspace.js` 通过。
- `node --test tests/extension-shell.test.mjs` 通过。
- `node --test tests/remake-core.test.mjs` 通过。

## 风险和需要注意的事项

- 这轮只做前端业务流程收口，不代表真实上游视频生成服务已经打通。
- 高级功能只是折叠，不是删除，后续仍可继续用。

## 执行后的实际结果

- 首页已从旧后台长页先收成普通用户主流程，随后继续收口成 `创建复刻任务` 向导。
- 已新增模特图上传入口，并保留产品图上传入口。
- 主页扫描完成后会自动命名模板并加入模板列表。
- 蒸馏细节、样本筛选、本地服务、批量任务和高级模板字段都已收进二级折叠区。
- 首页进一步收口后，主页链接已降成 `可选参考`，模板名称和蒸馏字段全部移入高级区。
- 最新口径已经不是“单主任务大卡片”，而是左侧步骤栏 + 右侧单步卡的创建任务向导。
- 生成区当前固定成四步：
  - 基础信息
  - 上传素材
  - 生成要求
  - 结果与提交
- 商品补充要求不再强制填写，普通用户只填商品名、提示词和产品图也能先生成。
- 历史记录已降成“最近生成”，默认只展示最近项目和第一条提示词预览，并新增 `一键提交去跑视频` 主入口。
- 页面进一步改成向导式消费端结构，减少首页上下过长和后台感过重的问题。
- 第二步上传产品图后，会自动按商品文件名和模板方向回填一版商品名、卖点和提示词草稿。
- 平台语言已收死成自动模式：TikTok 默认英文，抖音默认中文；高级区只保留口播风格/口音，不再允许手动改平台语言。
- 视觉方向已从普通后台风继续收口成更高端的工具感样式：深色主视觉、奶油色卡片、黑色主按钮和更精致的输入框层级。
- 最新一轮把视觉收成深绿 + 米白的向导式布局，首屏明确区分步骤导航、当前一步、结果提交和后台入口。
- 首页上半屏的大段说明已被砍掉，只保留主标题、自动能力说明和最小输入。
- 生成完成后，用户默认会优先看到提示词结果和提交入口，而不是后台结构细节。
- 代码检查通过；Chrome 当前已加载的扩展实例仍是旧缓存页，因此本轮页面核查以源码、测试和本地结构为准，待用户重新加载扩展后可直接看到新版前台。
