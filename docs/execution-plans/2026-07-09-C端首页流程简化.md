# 2026-07-09 C 端首页流程简化执行计划

## 用户原始需求摘要

用户已加载扩展，反馈 UI 有点乱，希望把蒸馏结果和无用功能隐藏掉，首页只保留最核心业务流程：填主页链接生成模板、上传模特图和产品图、填写提示词、设置生成条数，整体更像面向 C 端用户的网站。

## AI 对需求的理解

这轮不是补新蒸馏算法，而是把现有工作台收口成一套更简单的业务流程。核心目标：

- 首页默认只露出关键步骤，不再堆后台细节
- 主页链接扫完后自动生成模板并加入模板列表
- 提供模特图、产品图、提示词、提示词条数这些 C 端常用输入
- 历史记录保留，但蒸馏明细、筛片和服务配置收进二级区域

## 当前假设

- 主项目目录是`/Users/da/Desktop/飞蚁抖音复刻-Chrome扩展`。
- 用户允许直接执行，不需要再等确认。
- 这轮可以隐藏低频功能，但不删除底层能力。
- 不接真实生成、不改服务端。

## 计划修改范围

- `src/workspace.html`
- `src/workspace.css`
- `src/workspace.js`
- `tests/extension-shell.test.mjs`
- `README.md`
- `docs/execution-plans/2026-07-09-C端首页流程简化.md`

## 明确不修改的范围

- 不改 TikTok / 抖音主页抓取算法
- 不改复刻包核心输出结构
- 不接真实视频生成
- 不处理支付、额度、登录态抓取

## 执行步骤

1. 先核对当前页面结构和核心流程依赖。
2. 写失败测试，锁定简化版首页必须存在的核心入口。
3. 把首页重构成 4 步流程，隐藏蒸馏细节和低频功能。
4. 补模特图上传、主页扫描自动入模板列表、简化后的文案反馈。
5. 跑语法、测试和浏览器页面核查。

## 验收标准

- 首页默认显示 4 步：
  - 填主页链接生成模板
  - 上传模特图和产品图
  - 填提示词和生成要求
  - 生成提示词
- 蒸馏结果、筛片、批量任务、本地服务这些功能不再常驻首页主视觉。
- 页面存在模特图上传入口。
- 主页扫描完成后，模板会自动加入模板下拉列表。
- `node --check src/workspace.js` 通过。
- `node --test tests/extension-shell.test.mjs` 通过。
- `node --test tests/remake-core.test.mjs` 通过。

## 风险和需要注意的事项

- 这轮只做前端业务流程收口，不代表真实上游视频生成服务已经打通。
- 高级功能只是折叠，不是删除，后续仍可继续用。

## 执行后的实际结果

- 首页已重构成 4 步流程卡片，默认只保留 C 端用户最常用的输入和按钮。
- 已新增模特图上传入口，并保留产品图上传入口。
- 主页扫描完成后会自动命名模板并加入模板列表。
- 蒸馏细节、样本筛选、本地服务、批量任务和高级模板字段都已收进二级折叠区。
- 首页进一步收口后，模板名称已移入高级区，主页入口只保留平台、主页链接和模板列表。
- 生成区已改成更像网站表单：商品名、创作提示词、补充要求、生成条数和自动语言提示。
- 商品补充要求不再强制填写，普通用户只填商品名、提示词和产品图也能先生成。
- 历史记录已降成“最近生成”，默认只展示最近项目和第一条提示词预览，详细镜头与本地服务放进折叠区。
- 页面进一步改成更短的双列向导，减少首页上下过长的问题。
- 第二步上传产品图后，会自动按商品文件名和模板方向回填一版商品名、卖点和提示词草稿。
- 平台语言已收死成自动模式：TikTok 默认英文，抖音默认中文；高级区只保留口播风格/口音，不再允许手动改平台语言。
- 视觉方向已从普通后台风继续收口成更高端的工具感样式：深色主视觉、奶油色卡片、黑色主按钮和更精致的输入框层级。
- 本地静态预览和代码检查通过，当前首页已明显比旧版更像消费端网站，不再是后台工作台堆叠视图。
