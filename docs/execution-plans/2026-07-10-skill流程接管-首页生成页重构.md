# 2026-07-10 skill 流程接管：首页生成页重构

## 用户原始需求摘要

用户不想再继续靠零碎调样式推进，要求按更专业的分工方式接管：

- 先用产品经理视角重做首页结构
- 再从视觉和用户体验角度收口
- 最后由开发落实页面

## 我对本次任务的理解

这轮不再继续修单个空白、单个输入框，而是把 `生成` 页按正式工具产品的思路整体重构一版。

目标不是做花哨视觉，而是解决三个真实问题：

1. 首屏层级不够稳定
2. 主链路和辅助参数还没有彻底分层
3. 页面虽然已经比以前短，但还不够像成熟工具产品

## 当前假设

- 左侧 `生成 / 历史记录 / 蒸馏管理` 三栏目结构继续保留
- `历史记录` 和 `蒸馏管理` 当前拆分方向是对的，这轮主要重做 `生成` 页
- 继续复用现有生成逻辑、模板数据、图片上传逻辑和结果跳转逻辑
- 这轮重点是前台信息架构、HTML 结构和 CSS 密度，不改核心蒸馏算法

## 本次目标

把 `生成` 页改成一版更像正式工作台的 C 端任务页：

- 首屏更短
- 信息层级更稳定
- 主要操作更聚焦
- 辅助参数不再抢主流程注意力
- 桌面端不再出现明显“空壳区域”

## 产品经理结论

### 1. 普通用户真正要做的只有 4 件事

1. 选一个蒸馏模型
2. 上传产品图
3. 补一句创作要求
4. 点击生成

### 2. 其他字段不是主流程，只是“输出设定”

以下字段应该存在，但不该抢主流程注意力：

- 发布平台
- 视频比例
- 生成时长
- 口播语言
- 生成数量
- 可选主页参考
- 模特图

### 3. 自动卖点属于辅助信息

自动卖点不是首屏主输入，继续折叠是对的，但入口要更像“辅助展开”而不是第二主区块。

## 新首页结构定义

### A. 顶部只保留短标题，不堆解释

- 标题：`生成`
- 不再在首屏堆大段说明
- 解释性文案缩成轻提示

### B. 第一块：主任务区

这是整页最核心的一块，分成左右两列：

- 左侧：商品图 + 模特图上传
- 右侧：商品名 + 创作要求 + 生成按钮

这块的目标是让用户一眼知道：

- 图传这里
- 要求写这里
- 按钮点这里

### C. 第二块：输出设定

把这些字段收成一块独立的“输出设定”条：

- 蒸馏模型
- 发布平台
- 视频比例
- 生成时长
- 口播语言
- 生成数量

这一块不再冒充主任务区，而是作为生成前的参数设置。

### D. 第三块：可选参考

把这些辅助项独立为一条较轻的辅助区：

- 可选主页参考
- 平台语言自动切换
- 主页参考自动识别

### E. 第四块：辅助卖点

- 默认折叠
- 标题更轻
- 不和主任务区争抢视觉权重

## 视觉与体验规则

### 1. 首屏优先级

首屏优先顺序固定为：

1. 上传
2. 填要求
3. 生成
4. 再看参数

### 2. 卡片策略

- 减少连续多个同等级大框
- 主任务区可以稍强
- 输出设定和可选参考弱一级

### 3. 空白策略

- 不追求“铺满”
- 但避免出现大块无意义空白
- 空白只服务于层级，不服务于松散

### 4. 文案策略

- 面向普通用户
- 解释尽量短
- 不在首屏讲后台逻辑

## 计划修改范围

- `src/workspace.html`
- `src/workspace.css`
- `src/workspace.js`（只在结构变化必须同步时改）
- `README.md`
- 本执行计划文档

## 明确不修改的范围

- `src/remake-core.js`
- 蒸馏算法
- TikTok / 抖音抓取能力
- 本地视频任务服务协议
- 权限、额度、计费
- `历史记录` 和 `蒸馏管理` 的核心功能逻辑

## 执行步骤

1. 先重做 `生成` 页 HTML 结构，按“主任务区 / 输出设定 / 可选参考 / 辅助卖点”重排
2. 重写对应 CSS，让桌面端形成更稳定的两列主区 + 紧凑参数区
3. 只在必要时同步最小 JS，确保现有生成逻辑、上传逻辑和跳转逻辑不坏
4. 用真实页面截图核对桌面端和窄屏表现
5. 更新 README 和本执行文档，再做检查和 Git 提交

## 验收标准

- `生成` 页首屏明显比当前更清晰
- 用户一眼能看懂主动作是：上传、填写、生成
- 参数区不再显得比主任务区更抢眼
- 可选主页参考和自动卖点都退到辅助层
- 桌面端不再出现明显零散空白
- 生成逻辑、上传逻辑和生成后跳转 `历史记录` 仍正常
- 以下检查通过：
  - `node --check src/workspace.js`
  - `git diff --check`

## 执行后的实际结果

- `生成` 页已经重排成更稳定的 4 段：
  - 主任务区
  - 输出设定
  - 可选主页参考
  - 自动卖点折叠区
- 首屏最强操作已经改成：
  - 传商品图

- 输出设定不再挤在主任务区里抢注意力
- 左侧上传区不再被右侧输入区强行拉高，减少了无意义空白
- 现有上传逻辑、生成逻辑和生成后切到 `历史记录` 的行为继续保留
