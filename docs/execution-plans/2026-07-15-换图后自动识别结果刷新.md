# 2026-07-15 换图后自动识别结果刷新

## 用户原始需求摘要

用户要的是一条龙主链路可持续使用，不是第一次上传能用，后面换一批商品图就开始挂旧结果。

## AI 对需求的理解

代码审计发现商品图自动识别当前只会填“空字段”：

- 商品名
- 卖点
- 提示词草稿
- 场景
- 角色草稿

这会导致一个真实问题：

- 第一次上传商品图后，系统自动填了一版结果
- 用户又换了一批商品图重新上传
- 这些字段因为“不为空”，不会被新分析结果刷新

结果就是：后续生成项目时，表单里仍可能挂着上一批图的旧分析结果。

## 当前假设

- 本轮只补“自动填的旧值可被下一次自动分析刷新，手动改过的值尽量保留”。
- 不改 UI，不触发真实外部服务。

## 执行步骤

1. 补测试，锁定换图后自动识别结果会刷新旧自动值。
2. 修改 `src/workspace.js`，记录上一轮自动填充的字段快照。
3. 让下一次自动识别在“字段为空”或“字段仍等于上一轮自动值”时允许刷新。
4. 角色草稿也按同一原则处理；如果用户手动改过，则尽量保留。
5. 同步 `README.md`。
6. 跑检查命令并完成 Git 闭环。

## 验收标准

- 用户换一批商品图重新上传后，上一轮自动填的商品名、卖点、场景、提示词草稿和角色草稿可以被新分析结果刷新。
- 用户手动改过的字段不会被下一轮自动识别直接覆盖。
- `node --check src/workspace.js`
- `node --test tests/extension-shell.test.mjs`
- `git diff --check`

## 风险和需要注意的事项

- 不能把用户手动编辑过的字段无脑覆盖。
- 只处理自动填充刷新，不改生成业务逻辑。
