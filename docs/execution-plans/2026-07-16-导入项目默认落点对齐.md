# 2026-07-16 导入项目默认落点对齐

## 用户原始需求摘要

用户要把“选择蒸馏模型 -> 一键传图 -> AI 自动识别产品卖点场景 -> 生成提示词 -> 一键点击生成叙事图 -> 复制提示词图片一条龙”整条主链路继续收完整，导入历史项目时也要跟新生成项目一样直达最该看的位置。

## AI 对需求的理解

当前 `导入 JSON` 已经能恢复：

- 批次号
- 故事版状态
- workflow 状态

但默认落点还没有完全和“新生成项目”对齐：

- 新生成项目会切到 `历史记录`
- 右侧回到 `项目摘要`

而导入 JSON 当前只会提示“已导入”，不一定自动切到历史页，也没有主动把详情标签重置回摘要。

## 当前假设

- 本轮不改导入数据逻辑，只收默认落点。
- 导入完成后应直接切到 `历史记录`。
- 导入完成后右侧详情默认回到 `项目摘要`，跟新生成项目保持一致。

- `src/workspace.js`
