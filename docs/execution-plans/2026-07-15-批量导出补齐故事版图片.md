# 2026-07-15 批量导出补齐故事版图片

## 用户原始需求摘要

用户要的是完整一条龙，不只是能看到或单独下载故事版图片，而是导出结果时也能把提示词和图片一起带走。

## AI 对需求的理解

当前已经支持：

- 复制结果包文本
- 单张故事版图片下载
- 全部故事版图片下载

但 `批量导出项目` 仍然只会导出文本文件和图片链接清单，没有把故事版图片文件本体一起带走，所以导出链路还不完整。

## 当前假设

- 现有本地 `4328` 服务的故事版图片代理下载接口可以复用。
- 本轮不改导出包文本结构，重点是把导出动作补成同时下载故事版图片文件。

## 计划修改范围

- `src/workspace.js`
- `tests/extension-shell.test.mjs`
- `README.md`
- 本执行计划文档

## 明确不修改的范围

- 不改蒸馏算法
- 不改抖音 / TikTok 提示词语言逻辑
- 不改蒸馏管理页结构
- 不碰 `scripts/runtime/`
- 不碰 `src/workspace-flow-test.html`
- 不处理无关的 `src/workspace.css` 既有脏改

## 执行步骤

1. 先补测试，证明批量导出会继续把故事版图片本体一起下载。
2. 修改导出动作，复用本地图片代理接口，把故事版图片文件和文本文件一起下载。
3. 同步 README。
4. 跑相关检查命令并完成 Git 闭环。

## 验收标准

- `批量导出项目` 不再只导出文本和链接，也会下载故事版图片文件本体。
- 下载过程失败时给出清楚反馈。
- `node --check src/workspace.js`
- `node --test tests/extension-shell.test.mjs`
- `git diff --check`
- `git diff --cached --check`

## 风险和需要注意的事项

- 本轮继续依赖本地 `4328` 服务代理下载故事版图片。
- 必须处理部分图片下载失败时的反馈，不能一律报成功。

## 执行后的实际结果

- `批量导出项目` 现在会在导出文本文件的同时，继续下载故事版图片文件本体。
- 导出时会复用本地 `4328` 服务的故事版图片代理接口。
- 如果故事版图片只下载成功一部分，当前会明确提示成功数和失败数，不再一律报全部成功。
- README 已同步更新当前导出能力说明。
