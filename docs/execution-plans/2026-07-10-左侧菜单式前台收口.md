# 2026-07-10 左侧菜单式前台收口

## 用户原始需求摘要

用户要求前台进一步靠近腾讯云这类后台产品结构：

- 左侧固定菜单
- 右侧内容区
- 左侧栏目只保留：
  - 生成
  - 历史记录
  - 蒸馏管理

用户还明确要求：

- `生成` 仍然是最短流程
- `历史记录` 单独看结果
- `蒸馏管理` 单独管理模型和主页提炼

## 我对本次任务的理解

这轮不是继续优化单页表单，而是把前台切成三栏目的产品结构：

- 左侧：固定菜单
- 右侧：对应栏目内容
- 默认打开 `生成`

## 当前假设

- 继续复用现有生成逻辑、历史记录数据和蒸馏管理逻辑
- 只改前台页面结构和交互组织
- 不改蒸馏算法、API、权限和计费逻辑

## 计划修改范围

- `src/workspace.html`
- `src/workspace.css`
- `src/workspace.js`
- `tests/extension-shell.test.mjs`
- `README.md`

## 明确不修改的范围

- `src/remake-core.js`
- 真实外部服务调用
- 计费、权限、额度和后台协议

## 执行步骤

1. 把当前单页结构拆成左侧菜单和右侧内容区
2. 把生成表单保留在 `生成` 页面
3. 把最近结果移到 `历史记录`
4. 把蒸馏模型维护和主页提炼移到 `蒸馏管理`
5. 同步更新测试、README 和执行文档

## 验收标准

- 左侧固定出现 `生成 / 历史记录 / 蒸馏管理`

- `历史记录` 和 `蒸馏管理` 不再混在主生成页
