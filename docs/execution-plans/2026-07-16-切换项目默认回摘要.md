# 2026-07-16 切换项目默认回摘要

## 用户原始需求摘要

用户要把“选择蒸馏模型 -> 一键传图 -> AI 自动识别产品卖点场景 -> 生成提示词 -> 一键点击生成叙事图 -> 复制提示词图片一条龙”这条主链路继续收完整，切换项目时也不能让普通用户自己找当前项目的主信息。

## AI 对需求的理解

当前已经补了：

- 生成完成后自动切到 `历史记录`
- 并且右侧详情默认回到 `项目摘要`

但用户手动从项目库切换到另一个历史项目时，右侧详情标签仍然沿用上一个项目最后停留的标签。

这会造成一个轻微但真实的错觉：

- 人已经切到了项目 B
- 但右侧可能还停在项目 A 习惯看的 `批量任务 / 故事版 / 提示词` 视角

用户还是得再找一次当前项目的摘要入口。

## 当前假设

- 本轮不改项目数据和业务逻辑，只收切换时的默认视角。
- 用户从项目库切换到另一个项目时，右侧详情应默认回到 `项目摘要`。
- 手动再切换到其他标签的能力继续保留。
