# 2026-07-16 部分带走状态与建议收口

## 用户原始需求摘要

用户要继续把“选择蒸馏模型 -> 一键传图 -> AI 自动识别产品卖点场景 -> 生成提示词 -> 一键点击生成叙事图 -> 复制提示词图片一条龙”这条主链路收完整，不只是顺利时能跑通，半成功或异常时也要让普通用户看得懂下一步。

## AI 对需求的理解

当前代码已经支持 `一键带走部分完成`：

- 复制结果包失败
- 但文本导出继续完成
- 或故事版图片已下载一部分

不过前台还有两个口径缺口：

- 历史卡片状态会把 `一键带走部分完成` 直接算成 `已带走`
- `下一步建议` 没有针对“部分完成”单独说明，用户还得自己猜接下来是补复制、补下载，还是继续追视频批次

## 当前假设

- 主链路功能本身已具备，缺的是状态分层和建议文案。
- `一键带走部分完成` 应该被单独展示为 `部分带走`，不能和 `已带走` 混为一谈。
- `下一步建议` 需要根据是否已提交视频任务，分别告诉用户：
  - 文案 / 文件带走没完全收齐时，重新点一次 `一键带走全部结果`
  - 如果视频任务已提交，还要继续按批次号追踪视频任务
