# 2026-07-13 视频深蒸馏自动分析第二步

## 用户原始需求摘要

用户确认继续推进视频深蒸馏，不满足于只手动填写字段。希望直接读取本地文件夹里的视频，然后结合更稳的视觉模型，自动理解画面情绪、镜头推进、商品露出和卖点证明方式。

## 我对本次任务的理解

这次不是继续修 UI，而是把“视频深蒸馏”从样本承接推进到第一版自动分析：

1. 前台对本地视频做多帧抽样
2. 把抽样帧和视频元数据发给本地服务
3. 本地服务调用 Gemini 视觉能力生成结构化草稿
4. 前台把结果回填到每条视频的深蒸馏字段

## 当前假设

- 用户机器已有 `GEMINI_API_KEY`
- 现有本地服务命令可以改成仓库内脚本，避免继续依赖外部脚本副本
- 第一版自动分析先走“多帧图片 + 结构化输出”，暂不承诺整段视频直接上传视觉模型
- 本轮只做蒸馏管理相关链路，不动生成页和历史页主流程

## 执行步骤

1. 把本地服务脚本收回仓库
2. 增加深蒸馏分析接口和健康字段
3. 前台补“自动分析”入口和状态提示
4. 对本地视频做多帧抽样
5. 调本地服务拿结构化深蒸馏结果
6. 回填当前模型下的视频分析字段
7. 更新 README 和计划文档
8. 跑检查

## 验收标准

- 蒸馏管理页能触发“自动分析”
- 前台能对当前本地视频样本显示分析中状态
- 本地服务有深蒸馏分析接口
- Gemini 返回后会把结果写回对应视频字段
- `node --check src/workspace.js` 通过
- `node --check scripts/video-batch-service.mjs` 通过
- `git diff --check` 通过

## 风险和需要用户确认的事项

- 这一版自动分析仍然是“多帧抽样分析”，不是整段视频原文件直传模型
- 如果本机没有 `GEMINI_API_KEY`，自动分析只能失败或退回占位结果
- 由于浏览器安全限制，前台不能长期持有本地文件路径；关闭页面后如果要重跑分析，需要重新选择文件夹
