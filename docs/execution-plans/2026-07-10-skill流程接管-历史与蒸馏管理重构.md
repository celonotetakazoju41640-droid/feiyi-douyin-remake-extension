# 2026-07-10 skill 流程接管：历史与蒸馏管理重构

## 用户原始需求摘要

用户要求继续按专业分工方式推进，不再零碎手调，希望由产品经理先定结构，再由体验和开发落地。

## 我对本次任务的理解

上一轮已经把 `生成` 页改成主任务区优先的正式工作台。这一轮继续收 `历史记录` 和 `蒸馏管理`，让三个主栏目在产品层级上统一起来。

## 当前假设

- 当前 `历史记录` 功能链路基本可用，问题主要是结果、动作、详情和工具区混在一起
- 当前 `蒸馏管理` 功能很多，但还不够像后台工具页，字段和动作缺少分组
- 这轮可以优先重排 HTML 结构和 CSS 层级，不需要大改核心 JS 逻辑

## 本次目标

### 历史记录

把它改成“结果交付页”而不是“结果堆放页”：

- 首屏先看当前结果状态
- 高频动作放最前
- 详情和工具后置
- 最近生成独立承接

### 蒸馏管理

把它改成“模型后台”而不是“长表单”：

- 先看当前要做什么
- 再看基础字段
- 复杂控制拆进折叠区
- 样本筛片和参考视频继续保留，但不和基础信息抢主层级

## 新结构定义

### A. 历史记录页

首屏改成 4 段：

1. 当前结果概览
2. 高频动作条
3. 当前结果详情
4. 最近生成

导入、本地服务、镜头微调继续留在折叠区，退到辅助层。

### B. 蒸馏管理页

首屏改成 4 段：

1. 页面说明 + 主动作条
2. 模型基础信息
3. 模型深层规则折叠区
4. 主页蒸馏样本折叠区

参考视频、Clipcat 规则归入“更多控制”，不再和基础模型信息并排混放。

## 视觉与体验规则

- 历史记录强调“交付动作”
- 蒸馏管理强调“维护动作”
- 不再让所有板块都长得一样重
- 首屏说明尽量短，不讲多余背景
