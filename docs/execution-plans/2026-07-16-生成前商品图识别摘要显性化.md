# 2026-07-16 生成前商品图识别摘要显性化

## 用户原始需求摘要

用户要把主链路收成真正能直接跑的一条龙：

1. 选择蒸馏模型
2. 上传产品图
3. AI 自动识别卖点和场景
4. 生成提示词
5. 一键生成故事版
6. 复制提示词和图片直接带走

关键不是只把功能做出来，而是每一步都让用户一眼知道系统已经识别出了什么。

## AI 对需求的理解

当前上传商品图后，系统已经会：

- 自动回填商品名
- 自动回填卖点
- 自动回填场景
- 自动回填角色草稿

但生成前缺少一块简洁摘要，把这些识别结果前置展示出来。用户仍然要自己展开字段去核对，这和“傻瓜式主链路”还有差距。

## 当前假设

- 识别逻辑本身已经够用，当前缺的是展示层。
- 最小可行修复是在上传卡下方补一个识别摘要卡。
- 本轮不改识别算法、不改生成算法、不改外部服务。

## 计划修改范围

- `src/workspace.html`
- `src/workspace.js`
- `tests/extension-shell.test.mjs`
- `README.md`
- 本执行计划文档

## 明确不修改的范围

- 不改 TikTok / 抖音提示词语言逻辑
- 不改 `src/workspace.css`
- 不碰 `scripts/runtime/`
- 不碰 `src/workspace-flow-test.html`

## 执行步骤

1. 先补壳测试，锁定生成前上传区需要展示识别摘要。
2. 修改 `src/workspace.html` 和 `src/workspace.js`，在上传卡下方补“商品图识别摘要”。
3. 同步 `README.md`，写清生成前现在可直接看到卖点 / 场景 / 角色识别结果。
4. 跑相关检查命令。
5. 只提交本轮相关文件并 push。
