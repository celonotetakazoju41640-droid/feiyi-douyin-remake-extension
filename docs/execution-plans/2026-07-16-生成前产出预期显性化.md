# 2026-07-16 生成前产出预期显性化

## 用户原始需求摘要

用户要的不是零散功能，而是一条龙主链路：

1. 选择蒸馏模型
2. 上传产品图
3. AI 自动识别卖点和场景
4. 生成提示词
5. 一键生成故事版
6. 复制提示词和图片直接带走

每一步不仅要能跑，还要让普通用户在点击前就知道下一步会产出什么。

## AI 对需求的理解

现在生成前已经能看到：

- 当前模板 / 默认模型 / 平台
- 商品图识别摘要

但在真正点击 `生成并一键带走结果` 前，用户仍然看不到这次会直接产出什么，比如：

- 会生成几条任务
- 是否会带故事版
- 主按钮点下去后是不是会继续一键带走

这会让用户在点主按钮前仍然带着不确定感。

## 当前假设

- 现有生成逻辑本身已经成立，缺的是生成前的结果预期说明。
- 最小可行修复是在生成区补一个“生成后会得到什么”的摘要。
- 本轮不改生成任务结构、不改故事版逻辑、不改带走逻辑。
