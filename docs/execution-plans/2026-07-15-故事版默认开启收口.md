# 2026-07-15 故事版默认开启收口

## 用户原始需求摘要

用户要的是一条龙默认跑通：

1. 选蒸馏模型
2. 传图
3. 自动识别卖点和场景
4. 生成提示词
5. 一键生成叙事图
6. 复制提示词和图片带走

## AI 对需求的理解

当前故事版链路能力已经补齐很多：

- 可生成故事版任务
- 可自动等待结果
- 可下载故事版图片
- 结果区已有故事版主按钮

但默认故事版开关仍然是关着的，普通用户第一次使用时可能直接绕过故事版任务，导致整条链路默认不完整。

## 当前假设

- 故事版任务本地生成不额外触发外部成本，真正外部调用仍发生在用户点击故事版主动作时。
- 所以把故事版任务开关默认勾上，能让主流程更符合一条龙目标，而且不会在“生成项目”这一步直接偷跑成本。

## 计划修改范围

- `src/workspace.html`
- `src/workspace.js`
- `tests/extension-shell.test.mjs`
- `README.md`
- 本执行计划文档

## 明确不修改的范围

- 不改蒸馏算法
- 不改抖音 / TikTok 提示词语言逻辑
- 不改蒸馏管理页结构
- 不碰 `scripts/runtime/`
- 不碰 `src/workspace-flow-test.html`
- 不处理无关的 `src/workspace.css` 既有脏改

## 执行步骤

1. 先补测试，证明故事版开关默认开启。
2. 修改生成页默认值和清空后的恢复逻辑，让故事版回到默认开启状态。
3. 同步 README 说明当前默认主流程。
4. 跑相关检查命令并完成 Git 闭环。

## 验收标准

- 生成页里的故事版任务默认勾选。
- 清空项目后，故事版开关恢复到默认开启状态。

- `git diff --cached --check`

## 风险和需要注意的事项

- 这轮只改默认开关，不改故事版真实外部触发时机。
- 必须确保“默认开启”不会让用户在点 `生成项目` 时直接产生外部图片成本。
