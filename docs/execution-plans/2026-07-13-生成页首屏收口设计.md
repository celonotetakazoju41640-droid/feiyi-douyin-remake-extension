# 2026-07-13 生成页首屏收口设计

## 用户原始需求摘要

用户要求继续完善当前前台，并明确要求参考更专业的开发 skill 流程，不要再零碎地修一块算一块。

## 本轮目标

只收 `生成` 页首屏，让普通用户更快理解主流程，减少大留白和分散区块。

## 当前假设

- 用户要的是更像成熟 SaaS 后台的紧凑任务页
- 普通用户主流程只关心：传产品图 -> 补创作要求 -> 选输出设定 -> 生成/提交
- `主页参考` 和 `自动卖点` 是辅助，不应该抢主流程视觉优先级

## 设计结论

### 1. 页面结构

`生成` 页首屏改成一个主任务卡，不再拆成 4 个平级区块。

主任务卡内部顺序：

1. 上传产品图
2. 创作要求
3. 输出设定
4. 主页参考
5. 自动拆出的卖点（折叠）

### 2. 信息主次

- `上传素材` 和 `创作要求` 保持并排，是唯一的主操作区
- `输出设定` 收成紧凑横条，作为第二层配置
- `主页参考` 收成一条辅助输入，不再单独占一整大块首屏高度
- `自动拆出的卖点` 继续折叠，避免干扰首次操作

### 3. 视觉策略

- 减少区块之间的分割线和重复标题
- 压缩每块的上下留白
- 保留当前暖黄色系，不换视觉方向
- 保持左侧导航和其他两个栏目不重构，只做跟随式统一

## 修改范围

- `src/workspace.html`
- `src/workspace.css`
- `README.md`
- 本设计文档

## 不修改范围

- 不改生成逻辑
- 不改历史记录结构
- 不改蒸馏管理结构
- 不改本地服务和深蒸馏链路

## 验收标准

- `生成` 页首屏合并成一条更清晰的任务流
- 首屏高度明显缩短，辅助项不再抢主流程
- 页面功能不丢，交互入口不变
- `node --check src/workspace.js` 通过
- `git diff --check` 通过

## 执行后的实际结果

- 已把 `生成` 页首屏从多个平级区块收成一个主任务卡
- 已把 `主页参考` 收成辅助输入行，不再单独占一整块高度
- 已保留 `自动拆出的卖点` 折叠入口，避免干扰主流程
- 已同步更新 README 里的生成页口径
