# 2026-07-15 生成页主流程顺序对齐

## 用户原始需求摘要

用户要把“选择蒸馏模型 -> 一键传图 -> AI 自动识别产品卖点场景 -> 生成提示词 -> 一键点击生成故事版 -> 复制提示词和图片带走”这条主链路真正做成傻瓜式。

## AI 对需求的理解

当前代码口径和页面结构存在一处直接冲突：

- 主流程状态和按钮反馈会提示“先选蒸馏模型，再上传商品图”
- 但生成页实际 UI 仍把 `1. 上传产品图` 放在前面，把 `2. 选模板` 放在后面

这会让用户看到页面顺序和系统提示对不上，属于主流程级别的引导错误。

## 当前假设

- 本轮不改核心生成能力，只对齐主流程顺序和提示口径。
- 页面首屏应该按用户目标和系统状态口径，先让用户选模型，再传图触发自动识别。
- 对齐后，仍然保留“不填可选项也能生成”的原则。
