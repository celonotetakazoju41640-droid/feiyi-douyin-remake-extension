# 2026-07-13 深蒸馏结果接回生成链路

## 用户原始需求摘要

用户确认继续，希望不要停留在“深蒸馏能分析”，而是继续把这一轮做完整，让深蒸馏结果真正进入生成链路。

## 我对本次任务的理解

当前视频深蒸馏已经能读取本地视频并自动分析，但这些结果还没有真正影响生成页输出。  
本轮要把“模板里的深蒸馏结果”接回：

1. 生成时参考摘要
2. 蒸馏摘要
3. 候选版本
4. 批量任务提示词

让用户选定某个模板后，生成出来的提示词能明显带上该模板的视频画面结构，而不只是主页快蒸馏字段。

## 当前假设

- 当前模板里的 `deepDistillVideos` 已经能正常保存
- 用户仍然希望 C 端页面尽量简单，所以不新增复杂配置面板
- 第一版接回生成链路时，优先做“自动利用”，不增加用户理解负担

## 计划修改范围

- `src/remake-core.js`
- `src/workspace.js`
- `README.md`
- 本执行计划文档

## 明确不修改的范围

- 不改主页快蒸馏抓取逻辑
- 不改视频深蒸馏自动分析接口
- 不改历史记录页结构
- 不改单独的外部视频生成接口

## 执行步骤

1. 提炼模板里的深蒸馏视频摘要
2. 把深蒸馏摘要接入生成时的参考摘要拼装
3. 把深蒸馏字段接入蒸馏摘要和候选版本
4. 把深蒸馏字段接入批量任务提示词
5. 补 README 说明
6. 跑检查

## 验收标准

- 生成结果里的蒸馏摘要能体现视频深蒸馏结果
- 批量任务提示词能体现 0 帧起手、情绪曲线、镜头节奏、证明方式等字段
- 生成页在不增加额外复杂操作的情况下自动使用深蒸馏结果
- `node --check src/workspace.js` 通过
- `git diff --check` 通过

## 风险和需要用户确认的事项

- 本轮仍然是“把深蒸馏结果接回提示词链路”，不是直接生成成片
- 如果模板里没有视频深蒸馏结果，系统会自动退回只使用主页快蒸馏字段

## 执行后的实际结果

- 已新增模板级视频深蒸馏摘要提炼逻辑
- 生成时会自动把视频深蒸馏共性拼进 `referenceSummary`
- 蒸馏摘要现在会补：
  - 视频深蒸馏样本数
  - 0 帧起手倾向
  - 高频钩子
  - 高频情绪曲线
  - 高频镜头节奏
  - 高频证明方式
  - 高频收口方式
  - 画面推进共性
  - 视觉 DNA 共性
- 候选版本现在会按视频深蒸馏结果去区分节奏和证明方式
- 批量任务提示词已接入视频深蒸馏字段，不再只依赖主页快蒸馏
- README 已同步说明“深蒸馏结果自动进入生成链路”

## 验证方式

- 选择带深蒸馏结果的模板
- 进入生成页完成一次生成
- 检查蒸馏摘要、候选版本和批量任务文字
- 运行：
  - `node --check src/workspace.js`
  - `git diff --check`
