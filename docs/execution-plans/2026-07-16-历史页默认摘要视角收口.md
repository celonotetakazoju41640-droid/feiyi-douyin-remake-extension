# 2026-07-16 历史页默认摘要视角收口

## 用户原始需求摘要

用户要把“选择蒸馏模型 -> 一键传图 -> AI 自动识别产品卖点场景 -> 生成提示词 -> 一键点击生成叙事图 -> 复制提示词图片一条龙”这条主链路真正做成傻瓜式，生成完成后也应该直接落到最该看的完成页视角。

## AI 对需求的理解

当前生成完成后虽然会自动切到 `历史记录`，但右侧 `项目详情` 默认停在哪个标签页，仍然取决于用户之前看过哪个标签：

- 可能停在 `批量任务`
- 可能停在 `故事版图`
- 也可能停在 `提示词`

这意味着用户刚生成完项目时，未必第一眼就能看到最适合完成页的 `项目摘要`。

## 当前假设

- 本轮不改任何生成、故事版、提交或导出逻辑。
- 生成成功后切到历史页时，右侧详情应默认回到 `项目摘要` 标签。
- 用户后面手动切其他标签的自由仍然保留。
