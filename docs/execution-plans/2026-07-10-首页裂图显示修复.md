# 2026-07-10 首页裂图显示修复

## 用户原始需求摘要

用户反馈首页上传区出现“裂图”，未上传时就看到了坏图标和预览文字。

## 我对本次任务的理解

这轮只修首页上传区预览图在未上传状态下的错误显示，不扩到其他 UI 调整。

## 当前假设

- 上一轮为了修预览定位，给预览图补了 `display: block`
- 这条样式覆盖了元素原本的 `hidden` 隐藏效果
- 只要恢复 `hidden` 优先级，未上传状态就会回到干净的上传框

## 计划修改范围

- `src/workspace.css`
- 本执行计划文档

## 明确不修改的范围

- `src/workspace.html`
- `src/workspace.js`
- 历史记录
- 蒸馏管理
- 其他布局和文案

## 执行步骤

1. 修正预览图与 `hidden` 的样式优先级
2. 打开真实页面确认未上传时不再显示坏图
3. 再检查上传后预览仍然正常
4. 跑最小检查并提交

## 验收标准

- 未上传商品图和模特图时，上传框内不再出现坏图标或 alt 文案
- 上传后预览仍固定显示在对应格子里
- `git diff --check` 通过

## 执行后的实际结果

- 已移除导致 `hidden` 失效的显示覆盖
- 未上传状态下，商品图和模特图区域恢复成干净的占位上传框
- 上传后两张预览图仍会稳定显示在各自格子内部

## 验证方式

- 打开本地 `src/workspace.html` 观察首页首屏
- 先验证未上传状态不再出现裂图
- 再人工注入商品图和模特图，确认上传后预览仍正常
- 补跑：
  - `git diff --check`
  - `git status --short --branch`
