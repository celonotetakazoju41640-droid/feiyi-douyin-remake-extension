# 2026-07-15 生成页一键带走主动作

## 用户原始需求摘要

用户要的是一条龙主流程，不是先生成，再去历史记录里继续点第二个主按钮。

## AI 对需求的理解

当前已经有：

- 生成项目
- 结果区一键带走

但默认主流程仍然是两次主动作：

1. 生成项目
2. 一键带走全部结果

这还可以继续收成更像真正的一条龙。

## 当前假设

- 生成页应该直接给一个默认主动作：`生成并一键带走结果`
- 但为了不破坏已有使用习惯，仍保留一个次按钮：`只生成项目`

## 计划修改范围

- `src/workspace.html`
- `src/workspace.js`
- `tests/extension-shell.test.mjs`
- `README.md`
- 本执行计划文档

## 明确不修改的范围

- 不改蒸馏算法
- 不改抖音 / TikTok 提示词语言逻辑
- 不改蒸馏管理页结构
- 不碰 `scripts/runtime/`
- 不碰 `src/workspace-flow-test.html`
- 不处理无关的 `src/workspace.css` 既有脏改

## 执行步骤

1. 先补测试，证明生成页存在一键带走主动作。
2. 修改生成页按钮：
   - 主按钮：生成并一键带走结果
   - 次按钮：只生成项目
3. 生成后如果选择主按钮，会自动进入后续一键带走流程。
4. 同步 README。
5. 跑相关检查命令并完成 Git 闭环。

## 验收标准

- 生成页存在 `生成并一键带走结果` 主按钮。
- 用户点击一次主按钮，就能走生成 + 故事版 + 结果带走串行流程。
- `node --check src/workspace.js`
- `node --test tests/extension-shell.test.mjs`
- `git diff --check`
- `git diff --cached --check`

## 风险和需要注意的事项

- 旧的“只生成项目”路径要保留，避免影响只想先看结果结构的用户。
- 主按钮文案必须清楚，避免用户误以为会直接出成片。

- 旧路径 `只生成项目` 仍然保留。
