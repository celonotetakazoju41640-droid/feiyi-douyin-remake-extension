# 2026-07-10 首页上传框架与参数区重排

## 用户原始需求摘要

用户要求把 `生成` 页改成两种参考图混合后的结构：

- 图片上传区域按图 2 的素材框架来
- `蒸馏模型 / 条数 / 比例 / 秒数 / 平台选择` 按图 1 的参数区来

## 我对本次任务的理解

这轮不是新增业务能力，而是重做首页生成区的排版和控件组织方式。

目标是：

- 左边改成更像素材工作台的上传板
- 右边改成集中参数区
- 让“秒数”接回现有时长字段
- 让“比例”不只是界面显示，也进入当前生成结果和任务数据

## 当前假设

- 左侧菜单和历史记录、蒸馏管理结构继续保留
- 本轮主要改 `生成` 页
- 现有生成逻辑继续复用，不改蒸馏算法和外部服务协议

## 计划修改范围

- `src/workspace.html`
- `src/workspace.css`
- `src/workspace.js`
- `src/remake-core.js`
- `README.md`
- 本执行计划文档

## 明确不修改的范围

- `manifest.json`
- 历史记录数据结构的大范围重构
- 蒸馏算法
- 权限、计费和真实外部服务调用

## 执行步骤

1. 重排 `生成` 页为左侧上传区 + 右侧参数区
2. 把商品图和模特图上传控件改成素材板样式
3. 把模型、平台、条数、比例、时长改成集中设置区
4. 把时长控件接回现有 `targetDuration`
5. 把比例写入当前项目与批量任务数据
6. 跑检查并抓真实页面截图核对

## 验收标准

- 首页生成区视觉结构接近用户给的两张参考图
- 商品图 / 模特图上传不再只是普通长条文件框
- 首页能直接设置：
  - 蒸馏模型
  - 平台
  - 条数
  - 比例
  - 秒数
- 时长和比例不只是 UI，要进入当前生成数据
- 以下命令通过：
  - `node --check src/workspace.js`
  - `node --check src/remake-core.js`
  - `node --test tests/extension-shell.test.mjs`
  - `node --test tests/remake-core.test.mjs`
  - `git diff --check`

## 风险和需要注意的点

- 如果只改 UI 不接数据，比例会变成假功能，所以这轮必须把比例一起接进结果
- 这轮只处理首页生成区，不顺手重做历史记录和蒸馏管理的结构
