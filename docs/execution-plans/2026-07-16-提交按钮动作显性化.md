# 2026-07-16 提交按钮动作显性化

## 用户原始需求摘要

用户要把“选择蒸馏模型 -> 一键传图 -> AI 自动识别产品卖点场景 -> 生成提示词 -> 一键点击生成叙事图 -> 复制提示词图片一条龙”整条主链路做完整，细节也要考虑到位，不能让普通用户在关键动作上自己猜。

## AI 对需求的理解

当前历史区的主按钮之一仍叫 `提交生成`，但这个名字没有说清它到底会做什么。

真实动作其实是：

- 把视频批量任务发给本地服务
- 进入批次继续跑视频
- 返回批次号供后续追踪

对普通用户来说，`提交生成` 仍然太抽象，容易和“生成项目”“一键带走结果”混在一起。

## 当前假设

- 本轮不改提交逻辑本身，只补按钮和相关提示口径。
- 最小可行修复是把按钮、成功态、下一步建议、README 都统一成更直白的“提交到本地服务”口径。
- 不能影响现有批次号、故事版和一键带走逻辑。

## 计划修改范围

- `src/workspace.html`
- `src/workspace.js`
- `tests/extension-shell.test.mjs`
- `README.md`
- 本执行计划文档

## 明确不修改的范围

- 不改 `src/workspace.css`
- 不改 `src/remake-core.js`
- 不改提交接口协议
- 不改故事版底层任务逻辑
- 不碰 `scripts/runtime/`
- 不碰 `src/workspace-flow-test.html`

## 执行步骤

1. 先补测试，锁定提交按钮和相关口径应明确指向本地服务。
2. 修改 `src/workspace.html` 和 `src/workspace.js`，统一提交按钮文案和建议文案。
3. 同步 `README.md`，补充这个动作的真实含义。
4. 跑相关检查命令。
5. 只提交本轮相关文件。

## 验收标准

- 用户在历史区能直接看懂这个按钮是在“提交到本地服务”而不是重新生成项目。

- `node --check src/workspace.js`
