## 我对本次任务的理解

这轮只补两块真实使用闭环里的缺口：

- 当用户选到很空的模板时，生成页给出更轻量的新手引导
- 当用户已经生成出结果时，历史记录页先给出更短的当前结果摘要

## 当前假设

- 现有三栏结构和主流程已经基本确定，不再重做信息架构
- 当前最值得补的是“下一步该做什么”的即时提示，而不是继续加字段
- 这轮改动可以只落在前台页面和说明文档，不需要改核心生成逻辑

## 计划修改范围

- `src/workspace.html`

- `src/workspace.js`
- `README.md`
- 本执行计划文档

## 明确不修改的范围

- Chrome 扩展权限配置

## 执行步骤

1. 检查当前引导卡和结果摘要卡的代码接线
2. 打开真实页面核对两个场景的显示效果
3. 如果页面过挤、过长、文案过重，就做最小范围小修
4. 更新 README 和执行文档
5. 跑检查并完成 Git 提交、推送

## 验收标准

- 生成页在模板过空时会显示轻量引导，不会把主操作区挤乱
- 历史记录页在有结果时会先显示一块简短摘要，不喧宾夺主
- `node --check src/workspace.js` 通过
- `git diff --check` 通过
- Git 提交和推送成功

## 风险和需要用户确认的事项

- 本轮不触发真实外部视频生成，不产生额外外部成本
- 如果真实页面核对发现必须继续改结构，会先停在最小范围内处理，不顺手扩成新任务
