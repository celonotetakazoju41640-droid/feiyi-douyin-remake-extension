# 2026-07-14 生成页主流程重排

## 用户原始需求摘要

用户反馈当前“生成”页逻辑不清楚，点了之后看不懂到底有没有生成，页面层级也乱。希望把前台改成更傻瓜式的主流程，让人一眼看懂先做什么、按钮点下去会发生什么。

## 我对本次任务的理解

本轮不是修生成能力本身，而是把“生成”页收口成更直白的使用路径：

1. 先上传商品图
2. 再选择复刻模板
3. 然后直接生成项目

商品名、创作要求、故事版、主页参考这些内容不删除，但都要降级成可选补充，不能再压过主流程。

## 当前假设

- 现有生成逻辑已经可用，问题主要在前台结构和文案误导
- 用户希望默认更自动，不想被一堆字段拦住
- 生成后自动跳到“历史记录”的行为要保留，但文案必须说清楚

## 执行步骤

1. 重排生成页，把模板选择挪进主流程
2. 把商品名、创作要求改成“可选补充”
3. 把故事版中间层和主页参考改成后置展开区
4. 把“生成提示词”统一改成更符合真实行为的说法
5. 更新测试断言
6. 跑检查并完成 Git 提交上传

- 生成页首屏能直接看懂主路径：传图 -> 选模板 -> 生成项目

- 点击主按钮后的反馈文案明确说明已生成项目，并跳到历史记录
- `node --test tests/extension-shell.test.mjs` 通过
