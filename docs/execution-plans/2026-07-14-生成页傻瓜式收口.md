# 2026-07-14 生成页傻瓜式收口

## 用户原始需求摘要

用户反馈当前“生成”页虽然已经改过一轮，但还是看不懂、不会用，整体太复杂，用户体验很差。希望进一步收成真正傻瓜式的页面，不需要猜先点什么。

## 我对本次任务的理解

本轮不是修功能，而是继续压缩生成页复杂度，把首屏改成一眼能懂的主路径：

1. 上传商品图
2. 选择模板
3. 点击生成

其它字段和说明全部后置成可选内容，不能继续和主操作抢注意力。

## 当前假设

- 当前底层生成逻辑可用，主要问题仍然是前台层级和默认展示过多
- 平台、比例、时长、数量、草稿补充等都已经有默认值，可以后置
- 用户更想要“先跑起来”，不是“先研究表单”

## 计划修改范围

- `src/workspace.html`
- `src/workspace.css`
- `src/workspace.js`
- `tests/extension-shell.test.mjs`
- `README.md`
- 本执行计划文档

## 明确不修改的范围

- 不改蒸馏逻辑
- 不改生成结果结构
- 不改历史记录页逻辑
- 不改提交生成链路
- 不碰 `scripts/runtime/`
- 不碰 `src/workspace-flow-test.html`

## 执行步骤

1. 把首屏主流程压成“传图 + 选模板 + 生成”
2. 把平台、比例、时长、数量等移入可选设置
3. 把商品名、创作要求、卖点草稿并入可选补充
4. 收短提示文字，只保留操作型反馈
5. 更新测试和 README
6. 跑检查并完成 Git 提交上传

## 验收标准

- 用户进入生成页后，一眼能看懂先做什么
- 首屏只保留最少必要操作，不再像表单堆叠
- 不填写可选区时，也能明确知道可以直接生成
- `node --check src/workspace.js` 通过
- `node --test tests/extension-shell.test.mjs` 通过
- `git diff --check` 通过

## 风险和需要用户确认的事项

- 本轮会继续调整生成页结构和文案，但不改底层生成逻辑
- `src/workspace.css` 里仍有不属于本轮的未提交改动，提交时只暂存本轮相关内容
