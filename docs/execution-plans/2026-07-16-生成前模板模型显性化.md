# 2026-07-16 生成前模板模型显性化

## 用户原始需求摘要

用户要把主链路真正收成：

1. 选择蒸馏模型
2. 上传产品图
3. AI 自动识别卖点和场景
4. 生成提示词
5. 一键生成故事版
6. 复制提示词和图片直接带走

重点不是只让功能存在，而是每一步都让普通用户看得懂现在系统到底按什么在跑。

## AI 对需求的理解

现在生成后的历史卡片和结果摘要已经会显示 `模板 / 模型`，但生成前的第一步“选模板”里，用户仍然只能看到下拉框和模板视频重点，看不到这次默认会用哪个复刻模型。

这会造成一个真实盲区：

- 用户已经完成“选蒸馏模型”
- 但在真正点击生成前
- 还不知道后面任务默认会按哪个模型去跑

所以这轮要把模板 / 模型上下文前置到生成前，而不是继续等到生成后再补解释。

## 当前假设

- 模型绑定逻辑已经存在，缺的是生成前展示。
- 最小可行修复是在“1. 选模板”区直接给出当前模板和默认复刻模型。
- 本轮不改任务生成逻辑、不改外部服务。
