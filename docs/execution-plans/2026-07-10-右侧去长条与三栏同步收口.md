# 2026-07-10 右侧去长条与三栏同步收口

## 用户原始需求摘要

用户最新反馈是：

- 右侧不要再做成这么长条
- 生成页要更像中间窄工作区，不要横着铺太满
- 下面几个分类也一起处理
  - 历史记录
  - 蒸馏管理

## 我对本次任务的理解

这轮是在保留左侧三栏目结构不变的前提下，把右侧三个页面都改成更收口的内容宽度。

重点不是删功能，而是：

- 限制右侧主内容的最大宽度
- 减少整行横向拉满的感觉
- 让生成、历史记录、蒸馏管理三页都统一成更紧的后台工作区

## 当前假设

- 用户认可左侧菜单结构，不需要重做导航
- 这轮仍然只改前台排版，不改业务逻辑
- `历史记录` 和 `蒸馏管理` 的字段和功能先保留，只收排版宽度和分布

## 计划修改范围

- `src/workspace.html`
- `src/workspace.css`
- `README.md`
- 本执行计划文档

## 明确不修改的范围

- `src/workspace.js`
- `src/remake-core.js`
- 历史数据结构
- 蒸馏算法
- 外部服务调用
- 权限、计费和任务协议

## 执行步骤

1. 给右侧三个页面分别补更合适的内容宽度容器
2. 把生成页的基础设置、上传区和输入区改成更短的横向布局
3. 把历史记录页和蒸馏管理页也同步改成不横向铺满的工作区
4. 更新 README 和执行计划记录
5. 跑检查并看真实页面截图

## 验收标准

- 右侧不再出现明显整屏长条输入区
- `生成 / 历史记录 / 蒸馏管理` 三页都统一收口
- 页面仍然保持当前暖色和纯文字后台方向
- 现有交互和加载不受影响
- 以下命令通过：
  - `node --check src/workspace.js`
  - `node --test tests/extension-shell.test.mjs`
  - `node --test tests/remake-core.test.mjs`
  - `git diff --check`

## 风险和需要注意的点

- 如果收得太窄，蒸馏管理里的长文本区可能会显得拥挤，所以会按页面分别给宽度
- 这轮只处理宽度和排版，不顺手增删功能

## 执行后的实际结果

- 右侧主区已经从整屏长条改成中间窄工作区
- `生成` 页的基础设置、主页参考、上传区和提示词区都不再横向铺满
- `历史记录` 页和 `蒸馏管理` 页也同步套用了收口宽度
- 三个栏目当前使用不同的最大宽度：
  - 生成更窄
  - 历史记录中等
  - 蒸馏管理相对更宽
- 页面仍保留当前暖色和纯文字后台方向，没有改业务逻辑

  - 生成页已明显不再是长条
