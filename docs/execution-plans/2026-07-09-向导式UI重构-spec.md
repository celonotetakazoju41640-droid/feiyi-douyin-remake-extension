# 飞蚁抖音复刻 向导式 UI 重构 Spec

## 1. 用户原始需求摘要

用户对当前首页 UI 的核心反馈有三点：

- 页面块太大，首屏太撑，不像成熟工具。
- 当前结构还是在“拼页面”，不是成熟的“创建任务”流程。
- 希望参考成熟后台产品那种：左侧步骤栏，右侧只展示当前一步，底部有“上一步 / 下一步”。

用户明确给出的方向是：

- 参考图 2 那种创建任务式结构。
- 页面要更简单、更短、更收。
- 不再让复杂能力直接铺在首页。

## 2. 本次目标

把当前工作台首页重构成 `创建复刻任务` 向导，而不是普通长页表单。

目标不是只换视觉，而是同时收口：

- 信息架构
- 交互节奏
- 页面密度
- 默认暴露的信息量

## 3. 本次不做

- 不改 TikTok / 抖音主页抓取算法。
- 不改批量任务生成逻辑。
- 不接真实视频生成服务。
- 不改付费、额度、权限。
- 不新增后端。

## 4. 页面结构定义

### 4.1 总体结构

首页固定成三段：

1. 顶部轻量标题条
2. 左侧步骤栏
3. 右侧当前步骤内容卡

底部固定操作区：

- 上一步
- 下一步

第三步的下一步文案改成：

- 生成提示词

第四步不再显示下一步。

### 4.2 左侧步骤栏

左侧步骤栏只承担 3 个作用：

- 告诉用户当前在哪一步
- 告诉用户后面还有几步
- 允许用户点击切换步骤

左侧不放复杂说明，不放大段文案，不放业务细节。

固定四步：

1. 基础信息
2. 上传素材
3. 生成要求
4. 结果与提交

左侧还允许有一个很弱的提示块，只写系统默认逻辑：

- TikTok 自动英文
- 抖音自动中文

`官方高级后台` 在左侧底部作为次级入口，不进入主步骤。

### 4.3 右侧当前步骤卡

右侧永远只显示当前一步。

不允许首屏同时看到四步全部内容。

每一步内容卡遵循：

- 标题
- 一句简短说明
- 当前步所需字段
- 不出现跨步骤信息

## 5. 每一步字段边界

### 5.1 第一步：基础信息

只保留：

- 发布平台
- 对标主页链接（可选）

系统自动信息只做轻提示，不作为复杂表单：

- 当前语言自动判断
- 当前主页参考模式自动处理

第一步不要再放：

- 生成条数
- 产品图
- 模特图
- 商品名
- 卖点

原因：

- 用户刚进来时不该看到太多输入。
- 第一步的目标只是进入任务上下文。

### 5.2 第二步：上传素材

只保留两个上传块：

- 产品图
- 模特图 / 人物参考

要求：

- 产品图必填
- 模特图可选
- 两个上传块尺寸缩小，接近任务表单里的上传区，不再是海报式大模块

这一页不放：

- 商品名
- 创作要求
- 历史记录
- 结果区

### 5.3 第三步：生成要求

默认只保留：

- 当前商品名
- 生成条数
- 创作要求

自动卖点拆解不默认展开，改成折叠项：

- 展开自动拆出的卖点

原因：

- 这是辅助信息，不该和主输入并列抢注意力。

第三步的主动作是：

- 生成提示词

点击后直接执行当前已有生成逻辑，并跳到第四步。

### 5.4 第四步：结果与提交

第四步定义成完成页，而不是后台结果页。

上半区只保留高频动作：

- 一键提交去跑视频
- 复制提示词任务
- 下载 Markdown
- 下载 JSON

中间保留一个轻量历史区：

- 最近生成
- 第一条提示词预览

详细内容全部放折叠区：

- 详细结果
- 本地服务
- 导入 JSON
- 批量导出

第四步默认强调的是“交付动作”，不是“内部结构”。

## 6. 官方高级后台定义

`官方高级后台` 独立于主步骤。

里面保留：

- 模板管理
- 主页蒸馏
- 样本筛片
- 参考视频
- Clipcat 附加规则
- 本地服务相关入口

这些能力可以继续存在，但默认不进入主流程。

原则：

- 普通用户能完全不碰这里，也能完成生成。
- 这里只有运营和高级使用场景才展开。

## 7. 交互规则

### 7.1 下一步规则

- 第一步：直接去第二步
- 第二步：如果没有产品图，不允许进入第三步
- 第三步：点击“生成提示词”后执行生成，并跳到第四步
- 第四步：没有下一步

### 7.2 上一步规则

- 第一步不显示上一步
- 第二步可回第一步
- 第三步可回第二步
- 第四步可回第三步

### 7.3 左侧步骤点击

允许点左侧步骤切换，但要遵守最小约束：

- 可以回到已完成步骤
- 可以直接点到当前之前的步骤
- 不强制禁止点未来步骤，但未来步骤如果缺少必要输入，要给出轻提示

## 8. 视觉原则

本次 UI 不追求营销页感，追求成熟工具感。

视觉要求：

- 顶部短
- 左侧深色步骤栏
- 右侧浅色内容卡
- 模块圆角统一
- 字段密度收紧
- 每一步内容只保留关键输入

明确禁止：

- 首屏再出现大面积平铺卡片
- 一打开看到所有步骤
- 上传区做成海报级别大块
- 结果区默认展示大量后台详情

## 9. 实施顺序

1. 固定左侧步骤栏和右侧单步结构
2. 把四步字段按本 spec 重新分配
3. 收小第二步上传块
4. 把第四步改成完成页
5. 把官方高级后台彻底降级成次级入口
6. 跑壳层测试和生成链路测试

## 10. 验收标准

- 首页变成左侧步骤栏 + 右侧单步内容卡。
- 首屏不再同时展示所有步骤内容。
- 第一步只保留平台和可选主页参考。
- 第二步只保留产品图和模特图。
- 第三步默认只保留商品名、生成条数、创作要求；自动卖点折叠。
- 第四步先显示提交动作和最近生成，详细结果折叠。
- `官方高级后台` 不再混入主步骤。
- `node --check src/workspace.js` 通过。
- `node --test tests/extension-shell.test.mjs` 通过。
- `node --test tests/remake-core.test.mjs` 通过。

## 11. 当前风险

- 当前工作区已经有一版未提交的向导式 UI 改动，但还没有完全收口到这份 spec。
- 下一步应该停止继续凭感觉细修，改成严格按这份 spec 对照收口。
- Chrome 已加载扩展如果还是旧缓存实例，页面视觉判断会失真；验收时需要重新加载扩展。
