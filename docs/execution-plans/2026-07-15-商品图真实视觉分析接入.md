# 2026-07-15 商品图真实视觉分析接入

## 用户原始需求摘要

用户要的不是“按文件名猜一版”，而是上传商品图后，AI 真正根据图片内容自动识别：

1. 商品卖点
2. 适合的使用场景
3. 角色和演示方式
4. 提示词草稿

并继续接到后面的生成提示词、故事版和结果带走链路。

## AI 对需求的理解

当前代码里，上传商品图后的自动识别主要还是：

- 文件名
- 已填商品名
- 模板内容定位

本质是规则推断，不是真看图片内容。这和用户要的“AI 自动识别商品卖点场景”不一致。

## 当前假设

- 现有本地 `4328` 服务已经接通 Gemini 视觉能力，可复用同一套 JSON 分析模式。
- 前端上传图片后，先走真实视觉分析；如果本地服务不可用或结果不完整，再退回当前文件名规则兜底。
- 为避免用户在分析未完成时直接生成，生成按钮需要在分析期间暂时锁住。

## 执行后的实际结果

- 已在本地 `4328` 服务补上商品图视觉分析接口：
  - `POST /api/product-image-insights`
- 上传商品图后，前端会优先把图片内容发给本地视觉分析服务，不再只靠文件名推断。

  - 卖点
  - 场景
  - 角色草稿
  - 提示词草稿
- 服务异常时，前端会退回原有文件名 / 模板规则兜底，不阻断主流程。
- 分析期间生成按钮会暂时禁用，避免用户提前点击生成拿到半成品默认值。
- README 已同步更新当前真实能力。
