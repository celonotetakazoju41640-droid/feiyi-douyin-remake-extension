# 2026-07-16 生成页已生成状态口径收口

## 用户原始需求摘要

用户要继续把“选择蒸馏模型 -> 一键传图 -> AI 自动识别产品卖点场景 -> 生成提示词 -> 一键点击生成叙事图 -> 复制提示词图片一条龙”这条主链路收完整，不只是功能能点通，还要让前台每一步都说真话。

## AI 对需求的理解

当前主流程大部分能力已经具备，但生成页顶部 `主流程状态` 仍有一个容易误导的细节：

- 如果项目已经生成
- 且商品图识别摘要还保留着

页面可能继续强调“商品图已就绪，可以直接生成”，而不是先告诉用户“项目已生成，可以继续故事版、提交生成或一键带走结果”。

这会把“当前处于已生成阶段”和“还没生成、可以开始生成”混在一起。

## 当前假设

- 本轮不需要改功能链路本身，主要是修正文案优先级。
- 生成页里“已生成”阶段的状态提示，应优先于“商品图已识别、可以开始生成”的提示。
- 对于历史恢复但未重新上传商品图的项目，仍然要继续保留“先重新上传当前这轮商品图”的门槛提示。
