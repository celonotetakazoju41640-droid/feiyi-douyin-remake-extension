# 2026-07-10 skill 流程接管：交互体验精修

## 用户原始需求摘要

用户要求继续推进，不停留在静态 UI，而是继续把使用感做顺。

## 我对本次任务的理解

这轮只做交互体验精修，不改整体结构，不改业务逻辑。

重点解决：

- 上传后有没有即时反馈
- 生成前有没有明确的可用 / 不可用提示
- 历史记录切换项目时有没有更明确的当前状态提示

## 当前假设

- 当前页面结构已经足够
- 最大体验问题在于：用户做完动作后，页面反馈还不够强
- 这轮可以通过少量 HTML + CSS + JS 完成，不需要动核心生成逻辑

## 计划修改范围

- `src/workspace.html`

## 执行步骤

1. 给上传区补即时状态反馈
2. 给生成按钮补可用状态和禁用状态逻辑
3. 给历史记录项目切换补更明确的当前结果提示
4. 做真实页面截图核对
5. 更新 README 和执行文档，再做检查和提交

## 验收标准

- 用户上传产品图或模特图后，页面会立即显示更明确的素材状态
- 生成按钮在素材或要求没准备好时会明确不可用
- 历史记录切换项目后，当前结果提示更清晰
- `node --check src/workspace.js` 通过
- `git diff --check` 通过

## 执行后的实际结果

- 上传区已经补了即时状态 badge：
  - 商品图：未上传 / 已上传 X 张
  - 模特图：可选 / 已上传
- 生成按钮现在会按准备程度自动禁用或启用
- 商品名和创作要求输入过程也会实时影响按钮状态
- 历史记录切换项目后，当前结果提示会更明确告诉用户正在看哪个项目
- 历史记录空状态文案继续补强成更明确的下一步提示

## 验证方式

- 打开本地 `src/workspace.html`
