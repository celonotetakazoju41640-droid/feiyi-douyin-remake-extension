# 2026-07-16 提交后去向提示收口

## 用户原始需求摘要

用户要把整条主链路做成真正顺滑的一条龙，不只是按钮能点，还要每个阶段都让普通用户知道下一步去哪。

## AI 对需求的理解

当前 `提交生成` 成功后，前台已经会：

- 显示 `已提交到本地服务`
- 回写批次号
- 在历史页给出下一步建议

但提交后的建议还不够具体，没有把两条去向拆清楚：

- 视频任务已经进入本地服务 / 外部视频工具阶段
- 文案和图片整理仍然可以继续一键带走

这会让用户知道“提交成功了”，但仍不够清楚现在应该去哪里继续。

## 当前假设

- 提交逻辑本身已经成立，缺的是提交后去向提示。
- 最小可行修复是在历史页摘要和下一步建议里补充“视频去向 / 文案图片去向”。
- 本轮不改本地服务接口、不改任务结构。
